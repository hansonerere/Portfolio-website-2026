# 网站设计规范

本规范基于当前项目的页面结构、组件样式与交互语言整理，目标是为后续改版、内容更新与新增页面提供统一标准。

## 1. 设计定位

### 1.1 品牌气质
- 关键词：克制、现代、作品优先、编辑感、冷静、专业
- 核心感受：页面不主动“炫技”，而是让作品、影像与排版秩序本身构成高级感
- 视觉策略：大面积深色背景承托内容，卡片容器组织信息，图像与视频承担主要视觉张力

### 1.2 适用场景
- 个人作品集
- 创意工作室官网
- 品牌展示型网站
- 项目案例展示页

## 2. 视觉语言

### 2.1 色彩系统

当前站点的主色体系以深色中性色为主，建议后续统一沉淀为以下设计令牌：

| Token | 色值 | 用途 |
| --- | --- | --- |
| `bg.canvas` | `#0f0f0f` | 页面主背景 |
| `bg.surface` | `#212121` | 卡片、模块背景 |
| `bg.surface-hover` | `#2a2a2a` 或 `#424242` | 卡片 hover / 交互反馈 |
| `text.primary` | `#eeeeee` | 主标题、重点文字 |
| `text.secondary` | `#9e9e9e` | 正文、说明、辅助信息 |
| `border.subtle` | `#616161` | 分割线、弱边界 |
| `text.inverse` | `#0f0f0f` | 浅底按钮上的文字 |
| `bg.inverse` | `#eeeeee` | 浅色按钮、重点 CTA |

#### 色彩使用规则
- 背景优先使用 `bg.canvas`
- 信息容器统一使用 `bg.surface`
- 主文案用 `text.primary`，说明文案用 `text.secondary`
- 页面不使用高饱和品牌主色做大面积铺底
- 彩色只允许出现在图片、视频、渐变占位块或项目媒体中

### 2.2 字体系统

当前代码中强调字体为 `Red Hat Display`，建议将字体分工固定为：

- 展示字体：`Red Hat Display`
  - 用于站点名称、大标题、页头按钮、品牌感强的标题
- 正文字体：系统无衬线或与当前项目一致的 sans-serif
  - 用于段落说明、卡片描述、元信息

#### 字重建议
- `400`：正文
- `500`：小标题、按钮
- `700`：超大标题、品牌标题

### 2.3 字号层级

建议统一使用以下层级：

| 层级 | 建议尺寸 | 用途 |
| --- | --- | --- |
| Display XL | `clamp(3rem, 8vw, 8rem)` | About 页大标题、主视觉标题 |
| Heading L | `2rem - 3rem` | 大模块标题 |
| Heading M | `1.5rem - 2rem` | 卡片标题、区块标题 |
| Body L | `1.125rem` | 长段落正文 |
| Body M | `1rem` | 常规正文 |
| Body S | `0.875rem` | 元信息、辅助说明 |
| Caption | `0.75rem` | 注释、标签、年份说明 |

#### 排版规则
- 标题行高保持紧凑：`1.0 - 1.2`
- 正文行高保持可读：`1.5 - 1.7`
- 大标题尽量少换行，强调完整块面感
- 卡片内文字与图片区块之间保留明确呼吸感

## 3. 布局系统

### 3.1 总体结构

当前站点采用“双栏作品浏览 + 内容区域”的模式，建议固化为：

- 桌面端：左侧导航 / 项目列表 + 右侧内容区
- 平板端：保留双栏，但压缩左栏宽度
- 手机端：左右页面切换式结构，导航与内容全屏切换

### 3.2 间距系统

建议统一间距尺度：

| Token | 数值 | 用途 |
| --- | --- | --- |
| `space.2` | `8px` | 卡片间距、模块微间距 |
| `space.3` | `12px` | 标签与辅助信息 |
| `space.4` | `16px` | 常规内边距 |
| `space.6` | `24px` | 卡片标准内边距 |
| `space.8` | `32px` | 中型区块间距 |
| `space.12` | `48px` | 大模块间距 |
| `space.16` | `64px` | 页面级分区间距 |

#### 使用原则
- 卡片与卡片之间优先使用 `8px`
- 卡片内部统一以 `24px` 为基础内边距
- 页面级内容区保持稳定节奏，不随模块类型频繁跳变

### 3.3 圆角系统

建议统一：

- 卡片：`12px`
- 媒体容器：`12px`
- 大按钮：`999px` 圆角胶囊
- 小标签 / 浮层：`8px - 12px`

## 4. 组件规范

### 4.1 Sidebar 项目卡片

#### 结构
- 顶部元信息：分类 + 行为文案
- 中部主体：缩略图 + 标题 + 描述
- 底部信息：分隔线 + 年份

#### 状态
- 默认：`bg.surface`
- Hover：背景轻微提亮
- Active：浅色背景反转显示
- Loading：骨架屏替代真实内容
- Error：使用醒目的错误信息块，但不破坏整体深色风格

#### 规范
- 标题最多 2 行，描述最多 2 行
- 缩略图建议固定比例，避免列表高度波动
- 所有卡片高度尽量接近，保持列表秩序感

### 4.2 首页服务卡片

#### 结构
- 编号
- 标题
- 简介
- 视觉媒体或图片

#### 规范
- 统一使用纵向卡片
- 图片比例建议保持 `4:6`
- 标题不超过两行
- 描述控制在 2 至 4 行，避免卡片高度失衡

### 4.3 项目详情页

#### 模块顺序
1. Hero 影像
2. 项目基本信息
3. 文本内容区
4. 瀑布流图像 / 视频

#### 规范
- Hero 区必须有高质量封面图作为兜底
- 视频作为增强层，不应成为唯一内容载体
- 标题浮层保持半透明深底，保证可读性
- 图片墙优先保证节奏感，不追求绝对对齐

### 4.4 About 页面

#### 结构
- 顶部品牌头部
- 超大标题
- 2x2 信息卡片
- 底部 CTA
- Footer

#### 规范
- About 页不是“简历页”，而是“品牌介绍页”
- 文案强调方法论、背景、客户与联系方式
- 卡片允许部分使用纯色渐变占位，强化节奏变化

### 4.5 按钮

#### 主按钮
- 背景：浅色
- 文字：深色
- 形状：胶囊型
- 用途：返回项目列表、主 CTA

#### 次级按钮
- 背景：深色卡片色
- 文字：浅色
- 用途：局部操作、返回、重试

#### 规则
- 每个页面最多 1 个强主按钮
- 同一屏不要出现多个强竞争的 CTA

## 5. 媒体规范

### 5.1 图片
- 优先使用高质量、构图明确的作品图
- 缩略图与封面图要避免过多杂讯与复杂背景
- 尽量统一色调，降低页面整体噪音

### 5.2 视频
- Hero 视频建议无声自动播放
- 必须提供 poster
- 视频加载失败时必须回退到封面图
- 视频不应阻塞页面主体内容显示

### 5.3 占位图与渐变
- 仅在内容未准备完整时使用
- 渐变建议低饱和、单方向过渡
- 不要在一个页面中同时出现过多不同颜色渐变

## 6. 动效规范

当前项目动效风格偏轻量，建议继续保持：

### 6.1 基础原则
- 动效为“辅助阅读”，不是“视觉主角”
- 总时长尽量控制在 `200ms - 500ms`
- 以透明度和轻位移为主，少用复杂缩放与旋转

### 6.2 推荐动效类型
- 页面初载入：淡入 + 轻微上移
- 内容分段出现：stagger
- 图片进入视窗：渐显
- 卡片 hover：背景亮度变化 + 极轻微缩放

### 6.3 禁止项
- 不使用过度弹跳
- 不使用长距离飞入
- 不使用高频闪烁或持续运动背景

## 7. 响应式规范

### 7.1 移动端
- 导航与内容分屏切换
- 返回按钮固定在左上角
- 卡片字号略增，保证触控可读性
- 点击区域最小高度建议不低于 `44px`

### 7.2 平板端
- 保持双栏浏览逻辑
- 左栏宽度控制在可阅读范围内，避免挤压内容区

### 7.3 桌面端
- 作品内容占主要视觉空间
- 左栏信息密度可以更高，但不可拥挤

## 8. 文案规范

### 8.1 文案语气
- 简洁
- 专业
- 不夸张
- 偏编辑式表达

### 8.2 文案长度建议
- 项目标题：2 至 4 个词优先
- 项目描述：1 至 2 句
- 服务说明：不超过 3 行
- About 卡片正文：1 至 2 段为宜

### 8.3 命名规则
- 项目分类建议统一使用英文单数或固定体系
- 年份、地点、联系方式展示格式保持一致

## 9. 可用性与可访问性

### 9.1 可读性
- 正文与背景对比度需稳定
- 半透明浮层上的文字必须可读
- 辅助灰文字不应过淡

### 9.2 交互反馈
- 所有可点击卡片必须有 hover / active 区分
- 加载状态必须有骨架屏或明确提示
- 错误状态必须支持重试

### 9.3 媒体回退
- 图片失败时展示 fallback
- 视频失败时展示封面图
- 异步数据失败时不应让页面完全空白

## 10. 开发落地建议

为便于后续维护，建议下一步将这些规范沉淀到代码层：

1. 抽离设计令牌
   - 将颜色、圆角、间距、字号统一写入全局变量

2. 收敛类名与组件样式
   - 将常见卡片、按钮、标题样式抽成可复用组件或样式片段

3. 建立页面模板
   - 首页、项目详情页、About 页使用统一的 section 模板

4. 收敛动效时间
   - 统一页面入场、卡片 hover、图片显现的时长和 easing

5. 清理 Tailwind 配置
   - 缩小 content 扫描范围，避免无关文件进入构建扫描

## 11. 推荐的设计原则总结

- 作品永远优先于装饰
- 深色背景服务于内容，不喧宾夺主
- 卡片系统承担秩序感
- 留白比复杂装饰更重要
- 动效只负责增强节奏，不负责制造噱头
- 一个页面只表达一个清晰重点
